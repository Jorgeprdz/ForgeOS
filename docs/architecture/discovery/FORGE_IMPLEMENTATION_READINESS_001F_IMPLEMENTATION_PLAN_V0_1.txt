1. PURPOSE

This document defines the conceptual Implementation Plan v0.1 for Forge’s first implementation layer:

Evidence / Source / Truth Validation Layer

This plan does not implement code.

It defines what future implementation artifacts should exist, what each artifact should own, what each artifact must not own, what inputs and outputs are allowed, and what constitutional QA gates must be satisfied before code is written.

2. CONTEXT

Forge Constitutional Phase is locked.

ADR-001 through ADR-018 are canonical.

Implementation Readiness Layer 001 has produced:

- 001A — Source Ownership Registry
- 001B — Evidence State Vocabulary
- 001C — Truth Validation Result Contract
- 001D — Unknown / Stale / Conflict Handling Contract
- 001E — Cross-Domain QA Harness

001F translates those readiness contracts into a future implementation plan.

4. IMPLEMENTATION PHILOSOPHY

Future implementation must follow this order:

Evidence before interpretation.

Ownership before validation.

State before output.

Validation before recommendation.

QA before engine behavior.

Client First before revenue.

Human authority before automation.

Blocking output is better than inventing truth.

5. SCOPE

This plan covers only the future implementation of the conceptual layer that answers:

- Who owns this source or claim?
- What state is the evidence in?
- Is the source valid for this claim?
- Is the output allowed?
- What output strength is allowed?
- What warning is required?
- Is human review required?
- Is validation required?
- Should output be blocked?

8. ARTIFACT A — SOURCE OWNERSHIP REGISTRY

Purpose:

Identify conceptual ownership of source types, claim types, evidence types and domain outputs.

Based on:

FORGE_IMPLEMENTATION_READINESS_001A_SOURCE_OWNERSHIP_REGISTRY.txt

Owns:

- source ownership lookup
- claim ownership lookup
- domain ownership map
- allowed consumers
- prohibited reinterpretations
- governing ADR references
- owner unknown / owner conflict states

Does not own:

- evidence validity
- business truth
- product truth
- policy truth
- forecast truth
- compensation truth
- human judgment
- implementation behavior

Allowed conceptual inputs:

- source_type
- evidence_type
- claim_type
- consuming_domain
- domain_context

Allowed conceptual outputs:

- conceptual_owner
- ownership_status
- allowed_consumers
- prohibited_reinterpretations
- governing_ADR
- owner_unknown
- owner_conflicting
- consumer_allowed
- consumer_restricted
- reinterpretation_blocked

Blocked outputs:

- invented owner
- local fallback owner
- silent conflict resolution
- consumer reinterpretation as truth

QA gate:

If owner is unknown or conflicting, strong output must be blocked.

17. PRE-CODE QA CHECKLIST

Before writing code, answer:

- Which ADR governs this artifact?
- What truth does it own?
- What truth does it not own?
- What inputs may it consume?
- What outputs may it produce?
- What outputs are blocked?
- What degraded evidence states are handled?
- What cross-domain scenarios test it?
- What warning language is required?
- What human review states are required?
- What Client First risks exist?
- What Authority Boundary risks exist?
- What pressure, ranking or enforcement risks exist?
- What implementation behavior is explicitly not approved?

18. IMPLEMENTATION PROMPT REQUIREMENTS

When Codex/AGY is used for real implementation, the prompt must include:

- exact scope
- exact files allowed to create or modify
- no refactor unless required
- no feature expansion
- no UI
- no automation
- no scoring
- no ranking
- no enforcement
- no invented metrics
- required tests
- required validation
- summary in .txt format

19. CODE PHASE BLOCKERS

Implementation must stop if:

- owner is unclear
- evidence state vocabulary is missing
- validation result posture is undefined
- degraded evidence handling is missing
- cross-domain QA scenarios are missing
- output strength exceeds evidence
- unknown is hidden
- stale is treated as current
- conflict is silently resolved
- forecast becomes fact
- recommendation becomes mandate
- manager signal becomes enforcement
- productivity becomes human worth
- economic motivation becomes pressure
- Client First risk exists
- Authority Boundary risk exists
