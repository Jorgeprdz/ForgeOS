## Purpose

058A defines the separation contract for the Forge Alive static preview so desktop work can continue without contaminating the locked mobile baseline.

This document does not redesign desktop and does not reopen mobile. It records the current layer map, contamination risks, and target structure for a future reversible migration.

## Scope

In scope:

- `docs/static-preview/forge-alive/index.html`
- shared static preview CSS/JS inventory
- mobile 057D, 057E, 057F, 057G, 057J, 057L, 057M
- desktop 056Y and older desktop 056G7 surface
- Smart Widgets and command orb static-preview boundaries
- minimum CSS guard if safe

Out of scope:

- visual redesign
- CRM writes
- calendar creates
- message sends
- runtime provider execution
- browser storage
- schema changes
- deleting historical files

## Contamination Risks

1. `styles-desktop.css` starts with `@media (min-width: 768px)`, which overlaps tablet/mobile landscape territory. This is a known risk because the desired desktop boundary is `min-width: 901px`.
2. `index.html` contains desktop and mobile markup in the same `.phone-shell`.
3. Several mobile scripts run on page load and then self-guard with `matchMedia`; this is acceptable short term but should become entry-guarded.
4. `alfred-responsive-ui.js` is historically broad and can create mobile command/nav roots outside the clean 057D-057M line.
5. `command-bar-orb.js` is shared and moves visually between mobile and desktop slots through other scripts, so it must remain boundary-aware.
6. `.bottom-nav` is base markup and must be hidden on desktop.
7. `.forge-desktop-workspace-056y` and `.alfred-desktop-app-056g7` must never appear in mobile.

## Boundary Decision

Mobile applies only:

```text
(max-width: 767px), (max-width: 900px) and (orientation: landscape)
```

Desktop applies only:

```text
(min-width: 901px)
```

Shared files cannot contain viewport-specific layout except resets, tokens, and explicit boundary guards.
