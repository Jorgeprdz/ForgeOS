# Forge UI Design Line 001

Status: LOCKED / MOBILE-FIRST DESIGN DIRECTION

## Purpose

This document defines the visual and experiential line for Forge OS.
It is not a component specification. It is the north star that all Forge UI surfaces must respect.

Forge is a Sales Operating System. The UI must help a financial advisor or manager answer:

> Haz esto ahora, con este cliente, por esta razon.

## Non-negotiables

- Mobile first.
- Fewer clicks.
- Action over information.
- Human decision remains visible.
- Alfred is a concierge/intelligence agent, not a generic chatbot.
- Command bar is a primary action surface, not decorative search.
- Forge does not copy Apple, Samsung, Salesforce, or Sketch. Forge adapts useful patterns into its own identity.

## Visual Identity

The locked Forge palette is:

- Navy: base, calm, professional system surface.
- Gold: action, recommendation, Alfred active state.
- Blue/cyan: intelligence, signal, context, halo.
- Green: positive production or progress.
- Red: gap, risk, warning.
- White and blue-gray: text hierarchy.

## Product Feel

Forge should feel:

- premium;
- calm;
- decisive;
- trustworthy;
- operational;
- fast to act in;
- human-approved.

Forge should not feel:

- like a generic CRM;
- like an admin debug screen;
- like a consumer wellness clone;
- like mobile stretched into desktop;
- like a chatbot pasted onto a dashboard.

## Borrowed Patterns, Forge Translation

| External pattern | Forge translation |
|---|---|
| iOS / One UI material depth | Navy glass, blur, border, soft light |
| Samsung Health adaptive cards | Smart Widgets with action context |
| iOS tab bars | Forge bottom nav with premium states |
| iOS toolbars | Forge contextual top nav |
| Command palettes | Alfred command bar for business actions |
| Salesforce density | Desktop operational tables and panels |

## Mobile Direction

Mobile is for:

- quick decision;
- fast action;
- follow-up;
- reviewing next best action;
- opening previews;
- navigating to simple commercial tasks.

Mobile should prioritize:

1. safe state;
2. greeting/context;
3. Alfred recommendation;
4. plan of today;
5. Smart Widget;
6. compact opportunities;
7. KPIs;
8. bottom nav and Alfred orb.

## Desktop Direction

Desktop is for:

- quoting;
- policy upload/review;
- pipeline work;
- reports;
- charts;
- table operations;
- deeper Alfred workspace.

Desktop must be a professional system, not an enlarged mobile app.
