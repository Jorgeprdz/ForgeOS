# Forge Desktop Component System 001

Status: DRAFT / DESKTOP PATTERN LOCK

## Desktop Role

Desktop is the professional operating workspace for:

- quotes;
- policy upload;
- document review;
- reports;
- charts;
- table operations;
- manager views;
- deeper Alfred workflows.

Desktop must not be a scaled mobile view.

## Shell

Recommended structure:

- left sidebar;
- top command workspace;
- KPI strip;
- analytics/charts band;
- main operating table;
- secondary panels;
- right Alfred rail.

## Sidebar

Rules:

- compact;
- icon + label;
- active state clear;
- professional density;
- no oversized consumer cards.

## Command Workspace

Role: primary action shortcut.

Rules:

- near top;
- always easy to access;
- supports slash commands;
- preview-only by default;
- action chips below;
- results panel contextual.

## Tables

Rules:

- desktop can use tables;
- rows must have clear status and action;
- no mobile-style giant cards;
- no tiny unreadable columns;
- actions remain preview/review-first.

## Charts

Charts are required in desktop system direction.

Use cases:

- production;
- commissions;
- activity;
- 25 points;
- follow-up risk;
- team performance.

Charts must be useful, not decorative.

## Alfred Rail

Role: persistent reasoning and recommendation panel.

Sections:

- Haz esto ahora;
- Por que ahora;
- Limites;
- Comando recomendado;
- Feedback.
